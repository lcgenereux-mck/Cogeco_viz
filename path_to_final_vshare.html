Stage 1: Single Agent Testing
Description

Test the performance of a single agent

Owner

Each squad's PO

Environment

Agent development in AH-DEV

Activities
  • Automated testing: each squad writes its golden prompt-response dataset, with agent-specific metrics. Metrics batch run weekly
  • User research testing: focus groups and unmoderated testing on single-agent (non integrated to prod platforms)
  • Field testing: agent (non integrated to prod platforms) is given to some internal users, for feedback gathering
↓
GATE 1

PO certifies that their agent is ready to be integrated to workflow, with alignment from Crew lead.

↓
Stage 2: Workflow Level Testing
Description

Test the performance of an integrated workflow, connecting multiple agents

Owner

Crew lead

Environment

Workflow development in AH-DEV

Activities
  • Automated testing: single representative in the crew writes an integrated golden prompt-response dataset, for the workflow. Metrics batch run weekly
  • User research testing: focus groups and unmoderated testing on full workflow (non integrated to prod platforms)
  • Field testing: agent (non integrated to prod platforms) is given to some internal users (call center), for feedback gathering
↓
GATE 2

The staging environment in Agent Hub needs to be created.

Crew lead certifies that workflow is ready for integration to lower environment of customer facing platform.

Integration platform owners review testing results, approve integration and grant access to environments.

↓
Stage 3: Integration Testing
Description

Test the integration of an agentic workflow to its final interface

Owner

Crew lead

Environment

Agents as a service (AgentEngine/CloudRun, hosted in AH-Staging). Integrated to CCAIP staging environment

Activities
  • Automated testing: Continuous testing
  • User research testing: employees test chatbot in DialogFlow dev/staging environment
  • Safety testing: 3rd party ensures that agent is ready to ship to clients
  • Activate the AH-Staging environment
  • Integration of ADK agents (AH-Staging) to Dialogflow and Breezeline Web staging environments
OPEN QUESTION
  • Who handles integration of ADK agents to DialogFlow and Breezeline website?
  • What approvals are required to secure integration?
↓
GATE 3

Crew Lead, CAB review full test results and approve pre-prod release to subset of population.

↓
Stage 4: Pre-Prod Release to Subset of the Population
Description

Release the workflow to a subset of customers, in its final form

Owner

Crew lead

Environment

Agents as a service (AgentEngine/CloudRun, hosted in pre-prod environment). Integrated to CCAIP pre-prod environment

Activities
  • Automated testing: Continuous testing
  • User research testing: interpret customer feedback (read full transcripts of up/down vote)
  • Provision new pre-prod environment or create fork in production environment that can target just subset of the population (e.g. customers in a single state)
  • Promote Agents to pre-prod environment
OPEN QUESTIONS
  • What decisions will CAB take, what is their SLA?
  • What state or subset of the population will we favor in initial release?
↓
GATE 4

Crew Lead and CAB review full test results and approve prod release to full of population.

↓

Environment

Activities
  • Automated testing: Continuous testing
  • User research testing: interpret customer feedback (read full transcripts of up/down vote)
